Biocompute Object Example Draft 01

Antiviral Resistance Detection

Explanation

A biocompute object is a metadata record of an experimental instance; it has everything needed to reproduce a single in silico experiment. It contains references to all input data, all arguments of the underlying software application, and all version information of the executable program/programs used to run the bioinformatics computations.
Extending this idea, a complex web of separate objects is still a single instance where ALL the input, output, and parameters serve the same function as in a singular instance, and can therefore be joined with other similar objects. The biocompute object example used here is a combination of two separate computations that have been joined into one pipeline. This example can serve as inspiration for complex pipelines created by joining the appropriate nodes (inputs and outputs) of other biocompute objects. On this page we demonstrate how all information from one experimental instance can be easily captured in an extensible and human readable format.
This draft object was created using the protocol and results developed by Donaldson, et al. 2015.

Antiviral Resistance Detection

Object Instance:

The objectives of this biocompute object are to:
1) Identify baseline single nucleotide polymorphisms (SNPs), insertions, and deletions that correlate with reduced antiviral drug efficacy,
2) Identify treatment emergent amino acid substitutions that correlate with antiviral drug treatment failure, and
3) Determine whether the treatment emergent amino acid substitutions identified correlate with treatment failure involving other drugs against the same virus.

Type Definition:
Antiviral Resistance Pipeline

Pipeline Data Type

To create an instance (the biocompute object, or experiment record), the data type for that instance first needs to be defined. This is like writing a protocol before you conduct an experiment. A data type is a set of field and attributes related to specific object type, or a list of the input, output, and variables.
In a data type, the field-attribute collection is specified and a label is assigned for the collection.
In other words, we make a list of the different values and then define what can be put in those values. Then we name the list.

Type Definition:
Base Biocompute Type

Data Type

More complex and new types are created by the generation of extensible data types. Inheritance and inclusions are the two ways we use to extend a data format. In the case of inheritance, it is assumed that a more advance data type inherits all the fields from a simpler type and then extends the content only with additional field value pairs. As for inclusion, a particular field which was previously declared complex type includes all of the simpler data type. In this way we define the Base Biocompute Type with the fields necessary for all biocompute objects. This simple type is extended in complexity as needed by creating the pipeline type that is specific for each pipeline.

Type Definition:
Base Metadata Type

Data Type Hierarchy

A data type hierarchy contains multiple data objects linked to one another. The field attributes from one data object (the parent type) are inherited by one or many of the other data objects (the child types). In addition to the inherited field attributes, a given child type may have additional field attributes specific to that child type, as we saw above. In this example the Base Metadata Type is not only applicable for biocompute objects, but for any almost any other object type.

Object attribute Descriptions

_type

Object Type: This field defines the type of the object.

_inherits

Inheritance: This field indicates from which object or type the current object or type inherits properties from. Inheritance is when an object or type (class) is based on another object or type. This relationship between objects and types is what gives rise to inheritance.

_limit

Limited Field: This field indicates that only specific values are accepted for entry. If the "choice" is the sub field then the options listed wil be the ones available for selection.

_fields

Field: This is used in the defining of a data type.

_vital

Vital Field: This field indicates that this is a required field.

_plural

Plural Field: This field indicates that this is a field that can take multiple values. Multiple values are permitted, but not required.

_default

Default Value: This field indicates that there is a specified default value for this field. This is included to show that the default was not used in the instance

id

Object id: The object id is a unique identifying value assigned to each object. This value is used for all references to this object and it is not mutable. Unique object ids form the bases of object oriented programing. The unique identity of an abject allows that object to be treated

name

Name: The user-defined name of the object. This is mutable and has no value other than to the users. It can be any type of text.

description

Description: The creator-specified description of the object or field in a type. This is mutable and has no value other than to the users. It can be any type of text. This is where documentation type information should be included by the users. This will let other users know the function and purpose of the field/object.

format

File Format: Defines the output format of a file

filename

File Name: Defines the name of the output file

string

String: A string is a data type that stores a sequence of elements, typically characters or references to other objects.

list

List: A list is a data type that stores a number of elements, typically characters or references to other objects in a list.

datetime

Date and time: This is the class that returns an instance of time.

true

True Flag: This flag is applied to object fields that have a true/false value.

integer

Integer: These are numerical characters that have no fractional componenet, or whole numbers.

real

Real Numbers: Real numbers include all fraction and decimals, as well as the irrational numbers. If the numerical value is the result of, or going to be part of any operations it needs to be classified as a "real".
